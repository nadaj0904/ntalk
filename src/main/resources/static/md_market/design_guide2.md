





## 2. 폼 컴포넌트 규격 (`Guide02_Component`)

데이터 입력 및 상태 표시를 위한 기본 폼 컨트롤 디자인 규격입니다.

### 2.1. 체크박스 (CheckBox) 및 라디오 (Radio)
* **상태 구분**: `normal`, `mouseover`, `disabled` 3가지 유형으로 디자인상태를 구분하여 적용해야 함.
* **텍스트 간격**: 체크박스/라디오 아이콘 우측에 텍스트(`체크 01`, `선택 01` 등) 배치 시 정해진 마진 규격을 따름.

### 2.2. 탭 (Tab)
* **높이 (Height)**: 기본 `50px` (선택된 Tab 콘텐츠 영역 상단바 포함).
* **상태**: `selected/mouseover` (상단에 강조 줄 및 볼드 텍스트 적용), `normal`, `disabled` (흐림 처리).
* **부가 기능**: 좌우 너비를 초과하는 탭메뉴가 있을 경우, 우측에 `spinup/down button Pseudo`(좌우 꺾쇠 스크롤 화살표 버튼)를 구현해야 함.

### 2.3. 패널 그룹 (GroupBox & Div/PopupDiv)
* **GroupBox (그룹박스)**: `normal`, `disabled` 2가지 상태를 가짐. 외곽선 표기 형태 폼.
* **Div / PopupDiv**: 일반적인 사용은 Form을 로드해서 사용하거나 컴포넌트를 직접 정렬시키는 경우 활용됨. 디자인 요소가 별도로 필요할 경우 특정 `Class`로 지정하여 사용을 권고함.

### 2.4. 기타 상태 표시 요소
* **ProgressBar (프로그레스 바)**: 진행상태를 직관적으로 나타냄 (`normal` 상태 하이라이트 색상 / `disabled` 상태 그레이 색상).
* **ImageViewer (이미지뷰어)**: 단일 이미지를 담는 액자 형태 (`normal`, `disabled` 테두리 구분).

## 3. 버튼 컴포넌트 규격 (`Guide03_Component(btn+descript)`)

버튼은 목적과 영역에 따라 클래스를 분리하여 사이즈 변화 규칙을 따릅니다.

### 3.1. Common Button (일반 버튼) - 흰 바탕
* **기본 가이드**: 높이 `30px`. 기본 최소 너비 `50px`.
* **가변 길이 (Auto Width)**: 지정된 최소 50px을 기준으로 **한 글자가 늘어날 때마다 `12px`씩 가로폭 증가**. (예: `가나`=50, `가나다`=62, `가나다라`=74)
* **상태**: `normal`, `mouseover`, `pushed`, `disabled`.

### 3.2. CRUD Button (조회/저장 등 주요 액션 버튼) - 블루 바탕
* **기본 가이드**: 높이 `36px`. 기본 최소 너비 `80px`.
* **가변 길이**: 한 글자가 늘어날 때마다 **`13px`씩 가로폭 증가**.
* **상태**: `normal`, `mouseover`, `pushed`, `disabled` (일반 버튼과 동일 룰 적용, 강조 배경색 사용).

### 3.3. Grid Button (그리드 내부 액션 버튼)
* **기본 가이드**: 구조적으로 Common Button 형태와 속성이 동일. 너비 증가율 `12px/글자당`, 높이 `30px`.

### 3.4. Shuttle Button (이동/방향 버튼)
* **형태**: `29px * 29px` 정사각형 화살표 이동 버튼 (좌/우 방향, 처음/끝 방향, 위/아래 방향).
* **Class 지정**: `btn_WF_ShuttleL` (좌측), `btn_WF_ShuttleR` (우측), `btn_WF_ShuttleT` (상단), `btn_WF_ShuttleB` (하단) 등으로 클래스명 명시 필요.

### 3.5. Description (안내 문구 텍스트)
* 입력폼 우측이나 텍스트라인 하단에 제공되는 정보성 텍스트 규격.
* 일반 및 강조(붉은 텍스트)로 구성되며 앞부분에 `*` 별표를 동반함. (`* 중요 사항을 입력하여 주시기 바랍니다.`)

## 4. 데이터 그리드 가이드 (`Guide02_Component`)

대량의 데이터를 표현하는 그리드(표) 컴포넌트 규칙.

* **높이 규격**:
  * 단일 헤더 (Head Header) 높이: `30px`
  * 데이터 로우 (Data Row) 높이: `36px` 
  * 다중 헤더 (Multiple Head) 시: 뎁스별로 30px씩 누적 (예: 2단 헤더 = `60px`, `30+30`)
* **셀(Cell) 타입 지원**: 그리드 각 칸은 단순 텍스트 외에도 `Edit`, `MaskEdit`, `Combo(셀렉트)`, `Calendar(날짜)`, `Expand`, `Button`, `ProgressBar` 폼 요소를 임베드 가능하게 설계해야 함.
* **하단 Summary (합계/통계 로우)**: 그리드 제일 하단에 통계 로우(예: `합계 1,234,567`)를 표현하는 별도의 영역 존재 가능.
* **Grid Tree (트리 구조 그리드)**: 계층 구조를 나타낼 때 `첫번째 뎁스`, `두번째 뎁스` 등으로 아이콘(`+`, `-`)과 들여쓰기를 활용하여 표시함을 원칙으로 함.

## 5. 폼 레이아웃 (Search & Detail) 구성 (`Guide04_Component`)

검색 필드 구역 및 상세 정보 폼 구역을 그릴 때 적용하는 모듈화 치수 가이드.

### 5.1. Search (검색 영역)
상단 검색 컴포넌트 묶음의 행(Row) 개수에 따라 높이를 유동적으로 가져갑니다. 전체 총 너비는 보통 `1030px` 꽉 차게 디자인합니다.

* **최대 글자 노출**: 라벨(Label) 기본 최대 글자수는 6자이며, 6자 이상이면 라인을 분리하거나 두줄 처리.
* **단수에 따른 높이 변화**: 1단이 추가(다음 행)될 때마다 **높이가 `36px`씩 증가**함.
  * `1 줄일때` 단 높이: `52px` (패딩 여백 포함)
  * `2 줄일때` 단 높이: `88px`
  * `3 줄일때` 단 높이: `124px`
  * `4 줄일때` 단 높이: `160px`
* **입력 칸 너비 구분**: Label과 Input 넓이는 디자인 가이드 픽셀 룰 참조 요망 (기본 Label width는 아래 Detail 속성에 따름). 필수 폼의 경우 라벨 앞에 붉은색 `*` 를 부착함.

### 5.2. Detail (상세 정보 폼 컴포넌트)
하단 정보 입력 상세 테이블(Table/Grid 형식 폼). 전체 너비 `1030px` 기준, 테두리가 서로 1px 겹치도록(1px 겹침) 테이블 스타일로 라인이 맞물려야 함.

* **Row 높이**: 기본 `36px` 단위. 셀 병합이나 다중 컴포넌트에 따라 1단, 2단, 3단, 4단으로 구분 조합하여 사용.
* **Label Width Size 정의 (중요)**:
  * 최소 `8글자` 기준으로 가로 넓이를 넓힘 (기본 최소폭 `W140`).
  * 글자수(가나다라마바사아)가 **1자 늘어날 때마다 `12px`이 늘어나도록 유동 폭 적용**. (예: 9자-`W152`, 10자-`W164`, 11자-`W176`...)
  * 예시 외의 더 긴 글자가 들어오더라도, 글자가 잘리거나 가려지지 않도록 **가변적으로 width 값을 조정**해주어야 함.
  * 이 변경은 다른 줄의 동일 컬럼 라벨 너비에 영향을 주지 않음 (독립적 레이아웃 또는 Flex/Grid 구조를 통한 동적 처리 필요 권장).
