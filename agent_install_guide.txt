1. wiseT Agent 및 서비스  이해하기  
1.1. wiseT Agent 소개 
엠앤와이즈의  wiseT Agent 는 고객사  시스템에  설치되어  알림톡 /브랜드메시지 /문자 등의 대량의  메시지를  발송하는  
프로그램입니다 .  
wiseT Agent 는 고객사  시스템과  DB테이블  기반으로  연동되어  동작합니다 .  
고객사 DB에 발송 요청 인터페이스  테이블  생성 후, 고객사  시스템에서  인터페이스  테이블에  발송 데이터를  적재하면  
wiseT Agent 가 인터페이스  테이블내  발송데이터를  조회하여  메시지를  발송 처리합니다 .  
이후 발송 요청된  건에 대해 결과를  수신하여  인터페이스  테이블에  저장합니다 . 
 
wiseT Agent 는 아래 프로세스로  메시지  발송 및 결과 처리가  이루어집니다 .

2.4. 카카오톡  채널 생성 및 설정 
카카오톡  채널이란  카카오톡  계정을  기반으로  한 비즈니스용  카카오톡  아이디를  말합니다 .  
카카오  비즈니스  파트너센터  ( https://business.kakao.com/  )에 접속하신 후 [새 채널 만들기 ] 메뉴를  통해 채널을  개설할 
수 있습니다 . 
카카오  메시지  서비스인  알림톡과  브랜드메시지 를 발송하기  위해서는  카카오톡  채널이 반드시  필요합니다 .  
채널 생성 및 메시지  발송을  위한 추가적인  인증(비즈니스  인증 및 고객센터  정보 기입)에는 일련의  시간이  소요되므로  
사전에  생성 및 설정 작업이  필요합니다 . 
채널 개설을  완료 후, 실제 메시지  발송에  사용되는  발신프로필키를  엠앤와이즈를  통해 발급받아야  합니다 . 
발신프로필키를  발급을  위해서는  해당 카카오톡  채널의 비즈니스  인증 및 기타 추가 정보 설정이  반드시  필요합니다 . 
 
2.4.1.  비즈니스  인증 
비즈니스  인증이란  개설하신  카카오톡  채널이  브랜드 /사업체를  대표하고  있음을  별도의  심사 단계를  통하여  카카오  
팀에서  직접 확인해주는  서비스입니다 . 
카카오  비즈니스  파트너센터  ( https://business.kakao.com/  )에 접속하신  후 
[내 비즈니스 ] > [채널] > [비즈니스  심사]   메뉴를  통해 비즈니스  심사   를 신청합니다 .  
이후 카카오의  심사 승인 및 비즈니스  채널로  인증되어  알림톡  및 브랜드메시지 를 발송을  위한 발신프로필키  발급이  
가능합니다 .

해당 채널의  검색용  아이디를  확인 후 엠앤와이즈 ( help@mnwise.com  )에 전달하여  발신프로필키  발급을  요청합니다 .

3. DBMS 연동

DB(테이블  스페이스 )에 접근할  DB접속 계정을  만들고  DDL, DML 실행권한을  부여합니다 . 
3.2. 테이블  생성 및 확인 
엠앤와이즈가  전달한  SQL파일을  확인 및 실행하여  테이블을  생성합니다 .  
사용하는  채널에  따라 일련의  테이블들이  각각 생성이  되어야  합니다 .

3.3. 테이블  명세 
테이블  명세는  Oracle DBMS 기준으로  작성되었으며  다른 DBMS의 경우 아래 작성된  테이블  설명을  참고 바랍니다 . 
 
3.3.1.  알림톡  
알림톡을  보내기  위해 MZSENDTRAN 테이블에  데이터를  적재하면  Agent가 일정 주기로  발송 대상 데이터를  검색해  
발송합니다 . 
발송 후 결과 수신이  완료되면  MZSENDTRAN 테이블  데이터가  MZSENDLOG 테이블로  이동되며  두 테이블의  레이아웃은

비고1 SN(발송요청 ID) 값은 MZSENDTRAN, MZSENDLOG 두 테이블  모두 (이전 모든 발송 건에 대해) 고유한 값으로  
생성되어야  합니다 . 
비고2 P(푸시,PUSH) 타입 발송 방식은 대체 발송에  따른 메시지  중복 수신 문제를  해결한  방식으로  아래의  조건에  
만족하는  사용자에게  메시지를  전송하고  성공 처리합니다 . 
서버와 연결되어  있는 카카오톡  사용자  
발송 당일 가입한  사용자를  제외한  최근 7일(168시간) 내에 카카오톡을  사용한  사용자  
알림톡을  차단하지  않은 사용자  
조건에  만족하지  않는 경우 에러코드  NoSendAvailableException 를 결과로  응답합니다 . 
비고3 REQ_DT M(발송요청 일시)의 경우 현재 시간으로부터  24시간 이전 요청은  발송되지  않습니다 . 
 (예시) 
현재 시간이  20191210100000 일 경우 20191209100000 이후의  데이터만  발송 요청됨 
→ 20191209095959 이전 시간일  경우 발송되지  않음 
 
비고4 발송이  즉시 되지 않아도  되는 메시지의  경우 반드시  TR_TYPE_CD (발송우선순위 )를 9(배치발송 )으로 요청합니다 . 
대량 발송 건을 1(실시간발송 )으로 요청하면  이후에  인입된  실시간  발송 건이 앞서 인입된  대량 발송 건이 모두 
발송된  이후에  발송되기에  발송이  지연될 수 있습니다 .

비고1 MMS_BODY (LMS/MMS  발송 메시지 ) 컬럼에  발송 메시지를  직접 입력하는  경우 내부적으로  하나의  첨부 
파일(TXT)으로 인식하기  때문에  FILE_CNT (첨부 파일 수) 컬럼 값에 반영 필요합니다 .

4. wiseT Agent 설치 
서비스  사용 계약 후 설치 및 연동에  필요한  가이드  문서와  Agent 파일을 전달받습니다 . 먼저 연동할 DBMS에 테이블  
생성 스키마  SQL 파일을  실행을 진행한  후, 아래 순서대로  서버에  Agent 설치를  진행합니다 . 
4.1. 파일 업로드  및 압축 해제 
wiseT Agent 압축 파일을  서버에  업로드  후, 설치를  원하는  경로에  압축을  해제합니다 . 
a. Linux 서버일  경우 
- 업로드  파일 경로 예시 : /tmp/agent.tar  
- ${Agent 설치 경로} 예시 : /mnwise/agent   
① 업로드  파일을  Agent설치 경로로  복사 : # cp /tmp/agent.tar  /mnwise  
② Agent 설치 경로에서  압축 해제: # tar -xvf /mnwise/ agent.tar 
③ Agent 실행 파일 경로로  이동: # cd /mnwise/ agent/bin 
④ Agent 실행 파일 권한 변경: # chmod 755  
b. Windows 서버일  경우 
- 업로드  파일 경로 예시 : D:/tmp/agent. zip 
- ${Agent 설치 경로} 예시 : D:/mnwise/agent   
① 업로드  경로로  이동하여  파일(agent.zip) 을 복사 
② Agent설치 경로로  이동하여  붙여 넣기 후 해당 경로에  압축 해제

4.2. 설정 파일 수정 
(1) ${Agent 설치경로 }/conf/conf.properties 파일을  텍스트  에디터로  열어 로그 파일 경로 및 
연동DB설정 정보를  수정합니다 . 
고객사 ID와 TOKEN 정보는  엠앤와이즈에서  발급 진행하여  설정 파일에  기재하여  전달되므로  정보 기재가  누락되었는지  
확인하고  아래 파일 내용 예시에  노란 음영 부분을  수정합니다 . 
<!-- log 경로 --> 
log.dir= ${Agent 설치경로 }/log    ## 예) /mnwise/agent/log  
 
<!-- DB 설정--> 
jdbc.driver= [JDBC Driver]  
jdbc.url= [JDBC URL]  
jdbc.userId= [접속아이디 ] 
jdbc.password= [접속패스워드 ] 
 
<!-- Agent--> 
agent.id= [고객사 ID]           ## 엠앤와이즈에서  발급한  고객사 ID 정보 확인  
agent.token= [고객사 Token]     ## 엠앤와이즈에서  발급한  고객사 Token 정보 확인 
(2) ${Agent 설치경로 }/bin/base.sh( 리눅스 ) 또는 base.bat( 윈도우 ) 파일을  텍스트  에디터로  열어 
설치 경로 및 자바 경로를  수정합니다 . 
SERVER_ROOT= ${Agent 설치경로 }     ## 예) /mnwise/agent  
JAVA=" [JDK 설치경로 ]/bin/java"

5. 자주하는  질문 
Q. 에이전트  기동 시 failed: connect timed out 에러가  발생합니다 . 
방화벽  오픈이  정상적으로  되어있는지  확인이  필요합니다 . 
확인을  위해 에이전트를  설치하신  서버의  공인IP를 엠앤와이즈  서비스팀  메일로  전달하며  확인을  진행합니다 . 
내부적으로  명령 프롬프트에  telnet 명령어로  정상 연결 여부 확인하여  내부 방화벽  적용을  확인합니다 . 
telnet 182.162.100.234 8443  
 
Q. 에이전트  기동시  STOP-SIGNAL 에러가  발생합니다 . 
wiseT Agent 의 경우 1.9 JDK 버전에  대한 호환 이슈가  있어 추가적인  패치 작업이  필요합니다 .  
엠앤와이즈  서비스팀  메일로  로그 파일 및 자바 버전을 전달 부탁드립니다 . 
 
Q. Windows  서버에서  백그라운드로  실행되도록  하고싶습니다 . (윈도우  서비스  등록) 
윈도우  환경에서  에이전트  설치 시 서비스  등록 후 사용을  권장 드립니다 .  
서비스  등록 툴을 사용하시거나  명령 프롬프트를  통해 서비스 를 등록합니다 . 
※ 서비스  등록 후 실행 시, 기존 명령 프롬프트로 기동 된 Agent가 있다면  중지 후 서비스  실행을  부탁드립니다 . 
※ 서비스  등록 관련 상세 가이드 가 필요한  경우 help@mnwise.com 으로 요청 부탁드립니다 . 
 
Q. 에이전트  재기동  시 발송 요청 테이블에  남아있는  데이터가  발송되지  않습니다  
REQ_DTM( 발송요청일시 )가 현재시간으로부터  24시간 이내인  발송 건에 대해서만  발송 요청 처리됩니다 . 
해당 데이터를  즉시 발송을  원하시는  경우 REQ_DTM( 발송요청일시 ) 값을 현재 시간으로  업데이트  처리 부탁드립니다 .
